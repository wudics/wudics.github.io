最近发现无论是typora或者是其他markdown编辑器都不太满意，不是对阅读不友好，就是对编写不友好，所以琢磨出了一种编写和阅读都比较令人满意的方法，这篇文章就是告诉大家如何edge浏览器和vscode来编写和阅读markdown文件，营造一个舒适的markdown环境

Edge浏览器插件安装和配置

安装

插件名：markdown viewer

配置

首先需要配置的是允许渲染本地文件，这是必须的步骤，方法就是安装好插件后，在浏览器的插件图标上鼠标右键选中markdown viewer插件的管理扩展，或者选择markdown viewer右边的 “…” 按钮下的管理扩展，打开如下界面，给选项“允许访问文件URL”打勾

然后回到浏览器界面，鼠标左键选择插件图标，然后分别对主题、内容、编译器做配置

主题配置说明：

  • wide：是否打开宽屏模式

编译器配置说明：

  • gfm：默认开启即可

内容配置说明：

  • autoreload：修改markdown文件后自动重新加载和渲染
  • emoji：将emoji表情包的shortname渲染成img图片
  • mathjax：渲染数据公式
  • toc：渲染TOC目录导航 （这点对于这篇文章来说是必须开启的）

高级配置默认即可

vscode插件安装和配置

安装

插件名：markdown all in one

配置

默认配置即可

用vscode编写带有toc的md文章

假设我们的md文章带有标题，如下：

那么我们在vscode中，按F1键、或者按ctrl+shift+p快捷键、或者选择菜单中的“帮助->显示所有命令”，在弹出的输入提示栏中输入create table，在智能提示下选择“Markdown all in one: 创建目录”，即可完成TOC目录创建（PS：其实不创建TOC目录也是可以的，因为markdown viewer在渲染md文件时会自动创建）

用edge浏览md文章

右键选择*.md文件，设置打开方式为edge浏览器后双击打开md文件浏览，或者通过拖拽的方式，将md文件拖拽到浏览器界面打开，就可以用edge浏览器阅读md文件了

edge默认安装路径是C:\Program Files (x86)\Microsoft\Edge\Application\msedge.exe

左侧会显示有TOC目录导航
