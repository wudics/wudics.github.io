Introduction

Before writing an AMX Mod X module, you should have a good understanding of C and pointers.

жЁЎеқ—ејҖеҸ‘йңҖиҰҒзҶҹз»ғжҺҢжҸЎCиҜӯиЁҖе’ҢжҢҮй’ҲгҖӮ

AMX Mod X modules are significantly more advanced than normal AMX Mod modules. Here are a few main differences:

  1. API - AMX Mod X API is both backwards and forwards compatible. The actual attach interface and function prototypes will never change, what will change is what functions are available.

APIзҡ„и°ғз”Ёж–№ејҸеңЁдёҚеҗҢзүҲжң¬дёӯйғҪжҳҜзӣёеҗҢзҡ„пјҢдёҚеҗҢзҡ„жҳҜпјҢеңЁж–°зүҲжң¬дёӯеҸҜиғҪдјҡејғз”ЁдёҖдәӣAPIпјҢд№ҹеҸҜиғҪдјҡж–°еўһдёҖдәӣAPIпјҢдҪҶйғҪдёҚдјҡж”№еҸҳAPIзҡ„и°ғз”Ёж–№ејҸе’ҢеҸӮж•°зұ»еһӢгҖӮ

  1. Metamod - AMX Mod X modules do not attach to Metamod. They attach to a Metamod abstraction layer called FakeMeta - the functionality, however, is the same.

AMXXжЁЎеқ—дёҚдјҡзӣҙжҺҘи°ғз”ЁMetamodпјҢиҖҢжҳҜдјҡе…Ҳи°ғз”ЁFakeMetaпјҲMetamodзҡ„жҠҪиұЎеұӮпјү

  1. M/SDK - AMX Mod modules are built by manually writing interfaces and including a module header. With AMX Mod X, you simply add an MDK to your project. This MDK includes the memory manager, struct headers, core include references, configuration details, and .def alignment procedures (so you donвҖҷt need to make a .def file).

MDKеҸҜд»ҘзҗҶи§ЈдёәModule SDKпјҢдё“й—Ёз”ЁдәҺжЁЎеқ—ејҖеҸ‘зҡ„SDKпјҢеҢ…жӢ¬2дёӘжәҗз Ғж–Үд»¶пјҲamxxmodule.cppе’Ңamxxmodule.hпјҢиҝҷ2дёӘж–Үд»¶жҳҜдёҚйңҖиҰҒдҝ®ж”№зҡ„пјҢеҸӘйңҖиҰҒеңЁжЁЎеқ—йЎ№зӣ®дёӯеҢ…еҗ«.hж–Үд»¶еҚіеҸҜпјүе’Ң1дёӘй…ҚзҪ®ж–Үд»¶пјҲmoduleconfig.hпјҢеҸҜд»Ҙй…ҚзҪ®жЁЎеқ—зҡ„еҹәжң¬дҝЎжҒҜпјҢд»ҘеҸҠжЁЎеқ—зҡ„еҠҹиғҪпјүпјҢд№ҹе°ұжҳҜиҜҙеңЁејҖеҸ‘жЁЎеқ—ж—¶пјҢжҠҠamxxmodule.hе’Ңmoduleconfig.hеҠ е…ҘеҲ°йЎ№зӣ®дёӯеҚіеҸҜпјҲеңЁж–°зүҲжң¬дёӯпјҢamxxmodule.hдјҡиҮӘеҠЁеҠ иҪҪйЎ№зӣ®ж–Үд»¶дёӯзҡ„moduleconfig.hж–Үд»¶пјҢжүҖд»ҘпјҢеңЁдҪ зҡ„д»Јз ҒдёӯеҸӘйңҖиҰҒжҠҠamxxmodule.hж–Үд»¶includeиҝӣжқҘе°ұеҸҜд»ҘдәҶпјҢиҖҢдҪ зҡ„moduleconfig.hж–Үд»¶еҸӘйңҖиҰҒж”ҫзҪ®еңЁжЁЎеқ—йЎ№зӣ®зҡ„ж №зӣ®еҪ•дёӢеҚіеҸҜпјүгҖӮ

  1. AMX Mod X API uses an important function which will return a function pointer from the core based on the request:
    void *ReqFuncPtr(cost char *name);
    This lets the module retrieve any internally exported function from the AMX Mod X core, so you donвҖҷt have to recompile the module every time a new function is added to a struct.

    зӣёеҜ№AMXиҖҢиЁҖпјҢAMXXеҸҜд»ҘйҖҡиҝҮReqFuncPtrжқҘиҺ·еҸ–AMXXзҡ„ж ёеҝғеә“dll/soж–Үд»¶зҡ„еҮҪж•°жҢҮй’ҲпјҢеңЁд»Јз Ғз»“жһ„дёҠпјҢжҲ‘д»¬еҸӘйңҖиҰҒе®ҢжҲҗиҮӘе·ұmoduleжЁЎеқ—зҡ„ејҖеҸ‘е’Ңзј–иҜ‘еҚіеҸҜпјҢдёҚйңҖиҰҒйҮҚж–°зј–иҜ‘AMXXзҡ„ж ёеҝғд»Јз ҒгҖӮ

To begin, add вҖңamxxmodule.cppвҖқ and вҖңamxxmodule.hвҖқ to your project. You will never need to edit these.

Add вҖңmoduleconfig.hвҖқ. This will let you configure your module. There are a few important sections you should familiarize yourself with. Module Info: Edit the entries to personalize your module. Do not edit __DATE__, this is a macro.

еңЁmoduleconfig.hдёӯй…ҚзҪ®дҪ зҡ„жЁЎеқ—дҝЎжҒҜе’ҢеҠҹиғҪпјҢеңЁиҝҷйҮҢеҸҜд»Ҙй…ҚзҪ®жҳҜеҗҰејҖеҗҜMetamodпјҢд№ҹеҸҜд»ҘеңЁиҝҷйҮҢй…ҚзҪ®OnAmxx****зӯүиҝҷдәӣHookеӣһи°ғеҮҪж•°гҖӮ

Now youвҖҷre ready to go onto the next section to see how a basic module works.

Note - you can get these files from AMX Mod X CVS or by downloading the source code.
