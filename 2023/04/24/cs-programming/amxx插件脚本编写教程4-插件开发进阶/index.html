Tasks/Timers

Tasks，即定时器，能够让你在指定时间内执行你的回调函数，可以是一次性的，也可以是多次重复或无限循环的。它可以用来做一些有用的事情，例如，等待几秒后，使一些游戏对象销毁，又例如，循环执行一个任务。

task的运行方式是可以通过参数配置的，使用的函数是set_task()：

参数的含义如下：

  • Float:time - 时间间隔（秒），最小0.1秒
  • function[] - public声明的回调函数名称，string字符串类型
  • id - 任务ID号，需要自己指定
  • parameter - 数字数组类型，传递给task的参数数组
  • len - 参数数组的长度
  • flags - “a”表示循环指定次数；”b”表示无限循环；”c”表示地图更换倒计时后执行；”d”表示地图更换倒计时前执行
  • repeat - 如果flags是”a”，那么repeat的值就是循环次数

Menus

菜单是提供玩家选择的HUD信息。菜单代码的编写比较复杂，但是在cs1.6中是非常有用的，例如投票、管理员菜单等。

菜单Menus必须通过2个要素来注册。一个是keys，来告诉AMXX要注册多少个菜单选项。另一个是一个字符串，来标识唯一的菜单id号，这个此字符串将显示在要触发函数的每个菜单的开头位置。当你要显示一个菜单时，你可以只对某个玩家显示，也可以对多个玩家显示。当玩家通过按键选择对应的菜单选项时，选择的按键信息将会传递给注册菜单时的回调函数。

当出现一个%L时，我们至少要使用2个参数：

  • 最后的参数是多变参数，根据myplugin.txt中定义的value来决定，如%d、%s等。

Entities

FakeMeta

FakeMeta可以说是新一代的脚本编写规范。它可以让你在small脚本中使用metamod的功能。

  • Engine/DLL Calls：在HL引擎的命名空间中，有两种类型的函数，Engine 函数和DLL函数（DLL函数是HL引擎的game/mod的导出函数），两种函数可以在FakeMeta模块中通过engfunc()和dllfunc()这两个natives函数来调用。其参数将会被直接传入MetaMod中（所以要注意了，如果参数有问题，将可能会导致MetaMod崩溃，从而导致整个服务器崩溃）。
  • Engine/DLL Hooks：除了上边的native calls之外，我们还可以通过register_forward来拦截/重写forward函数的代码逻辑。你可以使用fm_return()和return PLUGIN_HANDLED来阻断后续代码执行。
  • 简易的实体修改：FakeMeta提供pev()和set_pev()两个native函数来替换HL引擎的entity__()函数，更简单地对实体的属性和行为进行修改。
  • Private Offset Hacking：Private offsets即实体私有属性偏移表，在HL引擎中为pvPrivateData。在使用时，需要很注意，不同的游戏它的pvPrivateData中的偏移值代表的含义是不同的，例如在cstrike中表示金钱，而在Natural-Selection中则表示资源，如果它的值设置错误了，很有可能会导致游戏崩溃。
